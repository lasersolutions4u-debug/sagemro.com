## 二、🔴 严重问题

### 2.1 LoginModal React Error #306（已恢复）

| 项目 | 内容 |
|------|------|
| 严重程度 | 🔴 阻塞 |
| 发现时间 | 2026-05-02 下午 |
| 持续时间 | ~3 小时 |
| 状态 | ✅ 已恢复 |

**根因分析**:

点击"登录/注册"按钮后，`LoginModal` 组件（通过 `React.lazy()` 懒加载）在 `Suspense` 中挂载时抛出 React Error #306：

```
Error: Minified React error #306
  args[]=undefined&args[]=

Stack trace:
  at Nc (vendor-react-Dk3RpLkG.js:8:80248)
  at Mu (vendor-react-Dk3RpLkG.js:8:115651)
  ...
  at Lazy (<anonymous>)
  at Suspense (<anonymous>)
  componentStack: at Lazy, at Suspense
```

**技术诊断**:
- Error #306 定义: "Should have a queue. This is likely a bug in React."
- `args[]=undefined&args[]=` 表示 `currentlyRenderingFiber` 为 null
- Hooks dispatcher 在组件渲染时未正确初始化
- 可能原因: (1) 懒加载组件使用了不同版本的 React 副本 (2) Module federation / chunk 分割导致 React 上下文丢失 (3) 构建产物中多个 React 实例冲突

**复现条件**: 100% 必现。任何用户点击侧边栏"登录/注册"均触发。

**影响范围**: 所有用户注册/登录完全不可用。页面不崩溃但弹窗静默失败。

**建议修复**:
1. 检查 `LoginModal` chunk (`LoginModal-CwwbnN5e.js`) 是否包含了独立的 React 副本
2. 检查 webpack/vite 的 `resolve.alias` 配置确保 React 单例
3. 检查 `React.lazy()` + `Suspense` 的 fallback 是否正确处理加载状态
