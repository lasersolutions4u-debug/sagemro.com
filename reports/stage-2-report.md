### 4.2 新增 Bug — 工程师注册静默失败

**发现日**: Day 10 (2026-05-07)
**严重程度**: 🟠 严重
**影响**: 第三位工程师注册被静默拒绝，导致当日工单 3 无法创建

**现场**:
```
注册后 data-testid 状态：
  'sidebar-login-button'        ← 表示未登录
  'register-engineer-button'    ← 表示表单还在
```
两个矛盾元素共存：系统认为用户未登录，但注册提交按钮仍然可见。

**根因推断**（两种可能指向同一类平台问题）:

| 推断 | 证据 | 可能性 |
|------|------|--------|
| 手机号已注册，后端拒绝但前端静默 | 第二轮执行（第一轮已注册该手机号）；`register_user()` 的 "已注册→登录" fallback 未触发 | 高 |
| 背景表单标签选中状态异常，后端校验失败静默 | SMS ⚠️、send-code API 未捕获 | 中 |

**核心问题**: 平台对注册失败的处理不完善 — 后端拒绝时前端没有展示可读的错误信息，用户无法判断失败原因。

**建议修复**:
1. 注册失败必须展示明确错误提示（当前静默失败，modal 僵死）
2. 已注册手机号应直接跳转登录，而非卡在注册表单
3. 提交按钮点击后应有 loading 状态，防止重复点击
4. 复查背景表单的标签组件选中状态判定逻辑
