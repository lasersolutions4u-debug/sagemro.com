# SAGEMRO Service OS 旧模型代码清理清单

日期：2026-06-07

## 标记原则

在 SAGEMRO Service OS 重构期间，旧的撮合平台、自由接单、工程师钱包、平台抽佣、托管支付等能力先不直接删除。

原因：

- 这些能力和现有工单、报价、工程师工作台仍有耦合。
- 直接删除可能破坏已有用户、测试数据、后台查看或内部运营。
- 需要先让自营服务申请、内部派工、服务报告、CRM 和 AI 线索闭环跑通。

代码中使用以下标记：

```text
SERVICE_OS_LEGACY
```

含义：

- 该代码属于旧撮合平台模型。
- 当前仅为兼容历史数据或内部过渡保留。
- 不应再作为新的用户侧产品能力宣传。
- 删除前必须完成本清单的替代能力。

## 待退役范围

### 1. 公开工程师注册

状态：已关闭。

- 前端公开注册入口已删除。
- Worker `/api/auth/register/engineer` 已返回 410。
- 内部工程师账号后续应改为后台管理员创建。

删除条件：

- 后台已有“创建内部工程师账号”能力。
- 无任何前端或测试依赖 `registerEngineer`。

### 2. 自由接单/拒单模型

状态：后台派工主路径已可用，工程师端旧确认/退回接口暂时保留。

旧能力：

- 工程师查看待接单工单。
- 工程师自主 accept/reject。
- AI 推荐工程师并触发待接单推送。

目标替代：

- 后台派工调度。
- 工程师只查看已分配服务。
- 管理员/客服确认后派给内部工程师。

已完成：

- 管理后台“服务运营”可选择内部工程师并派工。
- Worker 新增管理员派工接口 `/api/admin/workorders/:id/assign`。
- 工程师端旧 accept/reject 文案已改为“确认派工/退回调度”心智。
- 工程师任务列表默认只返回后台已派给自己的服务任务；旧 pending 队列仅在 `include_pending_legacy=1` 时兼容返回。
- 服务完成前必须先保存服务报告，避免空报告结案。

删除条件：

- 后台派工调度可用。
- 工程师端只展示已分配服务，不再拉取全量 pending 队列。
- 生产数据确认没有依赖 `/api/engineers/tickets/accept` 和 `/api/engineers/tickets/reject` 的自动化流程。

### 3. 工程师钱包/提现/平台抽佣

状态：公开/工程师侧接口已停用，旧结算函数和数据库字段暂时保留。

旧能力：

- 工程师钱包。
- 提现申请。
- commission_rate。
- 订单完成后按比例结算。

目标替代：

- 内部薪酬/劳务结算不对客户展示。
- 客户侧只看到 SAGEMRO 官方报价。
- 如需内部核算，应改为后台运营字段，不作为平台交易模型。

已完成：

- `/api/engineers/wallet` 和 `/api/engineers/wallet/withdraw` 返回 410，不再作为产品能力开放。
- 工程师设置页不再调用钱包接口，不展示钱包/保证金。
- AI 摘要协议从 `wallet_query` / `[payment_pending]` 改为 `account_or_settlement` / `[service_followup]`。
- 客户验收/评价完成后，不再向工程师推送钱包入账通知；底层结算函数仅作历史兼容。

删除条件：

- 真实收款/对账方案确认。
- 内部工程师薪酬或服务商结算方案确认。
- 支付弹窗不再表达托管/平台分账。
- 历史 `engineer_wallets` / `engineer_withdrawals` 数据是否保留归档已决策。

### 4. 服务报告 / SOP

状态：基础闭环已完成，后续可增强照片、客户签收和标准检查项。

已完成：

- 复用 `work_order_repair_records` 作为服务报告基础表，不新增 D1 migration。
- 前端“Repair Record”升级为“Service Report”口径。
- 服务报告包含客户现象、根因诊断、服务动作/后续建议、配件使用和工时。
- 工程师标记服务完成前，前后端均要求已有服务报告内容。
- 服务报告 SOP 已提示现场照片/验收文件走现有 Attachments 标签上传。

后续增强：

- 报告中关联现场照片/视频附件。
- 客户确认服务报告或电子签收。
- 按设备类型提供更细 SOP 检查项模板。

### 5. “工单”命名

状态：底层暂保留，用户侧逐步改名。

旧命名：

- Work Order / 工单

目标命名：

- Service Request / 服务申请
- My Services / 我的服务
- Service Report / 服务报告

删除条件：

- API、DB、后台和前端命名重构成本评估完成。
- 不影响历史数据。

### 6. 旧 AI Prompt 中的平台撮合角色

状态：已完成主 prompt 改造，历史兼容词仍保留在旧文档和少量兼容注释中。

旧内容：

- AI 作为撮合平台助手。
- AI 帮客户匹配工程师接单。
- AI 向工程师提示待接单工单。

目标替代：

- AI 是设备诊断与服务 intake 助手。
- AI 为 SAGEMRO 客服/调度提供结构化摘要。
- AI 不直接对外承诺工程师匹配。

删除条件：

- 新 6 个 AI 工具 prompt/schema 落地。
- Worker 系统 prompt 更新并通过测试。
- SummaryProtocol golden set 已切换到 Service OS 待跟进语义。
