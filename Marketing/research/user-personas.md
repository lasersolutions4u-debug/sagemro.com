## 一、主要用户画像

### 画像一：张厂长 🏭

| 维度 | 描述 |
|------|------|
| **身份** | 中小型钣金加工厂老板/厂长 |
| **年龄** | 35-50 岁 |
| **工厂规模** | 20-100 人，设备 3-15 台 |
| **所在地** | 长三角/珠三角/山东/河北 工业聚集区 |
| **典型一天** | 盯生产进度、接客户电话、处理设备突发故障、算成本 |
| **设备故障时** | 打开百度搜"XX故障怎么修"→ 可能打给认识的师傅 → 如果修不好就停工损失 |

**核心痛点：**
- 设备出故障耽误交货，损失大
- 不信任陌生维修师傅（怕被宰、怕修坏）
- 厂家售后太贵太慢（出保后尤其痛苦）
- 对各种设备缺乏系统保养知识（买来就用，坏了再修）

**内容消费习惯：**
- 早晨/晚上刷微信
- 遇到问题百度搜
- 偶尔看抖音/快手（推荐算法推的设备维修类视频）
- 不太用知乎和小红书

**触达方式：** 百度 SEO > 微信公众号 > 抖音/快手 > 行业微信群

**决策心理：**
> "我是搞生产的，不是搞设备的。最好有人能告诉我这是什么问题、要花多少钱、找谁能修好。"

| 维度 | 描述 |

**触达方式：** 知乎 > 百度 > B站 > 公众号 > 行业论坛

**决策心理：**
> "我需要的是准确的诊断方向和技术细节，不是泛泛的客服。给我干货，我自己能判断。"

> "有平台帮我接单、帮我定价、帮我背书当然好。但抽成不能太高，结算要靠谱。"

## 二、用户旅程地图

### 客户侧：从发现问题到解决问题

```
阶段 1：发现故障
  情绪：焦虑（生产停了）
  行为：观察设备、询问操作工
  触点：设备报警信息

阶段 2：自行排查
  情绪：尝试解决问题
  行为：百度搜索"XX故障怎么办"
  触点：搜索结果页 → 我们的技术文章

阶段 3：获取专业帮助
  情绪：需要可靠建议
  行为：点击进入 sagemro.com，与 AI 小智对话
  触点：AI 聊天窗口

阶段 4：决定下一步
  情绪：对比方案
  行为：AI 给出诊断结论 → 判断是否需上门
  触点：AI 推荐工程师/报价

阶段 5：服务交付
  情绪：期待问题解决
  行为：工程师上门维修
  触点：工单消息区、微信通知

阶段 6：持续关系
  情绪：满意/不满
  行为：评价服务、后续保养咨询
  触点：AI 小智定期关怀、公众号内容
```

### 工程师侧：从找活到接单

```
发现平台 → 注册提交背景 → AI 推荐工单 → 接单 → 核价 → 服务 → 评价 → 提现
   ↑                                                                    ↓
 抖音/百度引流                                                    等级提升/更多工单
```

## 三、面向用户的内容差异化

| 用户 | 内容关注点 | 语气 |
|------|-----------|------|
| 张厂长 | 能不能快速解决？多少钱？靠不靠谱？ | 直接、有底、不废话 |
| 李工 | 技术细节、参数、原因分析 | 专业、具体、有逻辑 |
| 王师傅 | 订单机会、平台抽成、结算方式 | 实在、透明、不画饼 |

**核心原则：对厂长说人话，对工程师说技术话，对师傅说实在话。**
