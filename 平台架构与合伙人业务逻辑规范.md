# 平台架构与合伙人业务逻辑规范 (V1.0)

## 1. 核心业务定位

- **行业领域：** 钣金加工（垂直于激光切割机维修）。
- **商业模式：** 总部（SAGEMRO）总包 + 区域合伙人（承包制）执行。
- **技术核心：** [小智智能助手](https://sagemro.com/) 提供的数字化 SOP 与专家知识库。

## 2. 法律与主体关系

- **非雇佣关系：** 总部与合伙人之间为 **B2B 服务外包/合作关系**。
- **合伙人主体：** 注册为个体工商户或小微企业，独立承担法律与安全责任。
- **总部角色：** 负责获客、资金结算（代收全款）、开具增值税专用发票、品牌授权、技术大脑。

## 3. 系统实体模型 (Data Entities)

### A. 合伙人 (Partner)

- `partner_id`: 唯一标识。
- `level`: 技术等级（初级/专家/核心专家）。
- `service_area`: 承包的地理区域编码（如：江苏省-苏州市-吴中区）。
- `deposit`: 保证金余额（用于违约或事故赔付扣除）。
- `skills`: 擅长设备列表（激光、折弯、焊接等）。

### B. 工单 (Work Order)

- `order_id`: 唯一标识。
- `client_info`: 客户工厂信息（开票抬头、机器位置）。
- `fault_description`: 故障描述 + [小智](https://sagemro.com/) 初诊预判结果。
- `status`: 待匹配 / 已接单 / 施工中 / 待验收 / 已结算。
- `sop_checkpoints`: 数字化 SOP 检查点（合伙人必须拍照/填报的节点）。

### C. 财务结算 (Finance)

- `total_amount`: 客户支付的全额。
- `commission_rate`: 平台抽佣比例（基于合伙人等级）。
- `payout_amount`: 合伙人实得服务费。
- `tax_deduction`: 代扣代缴的个税/管理费。

## 4. 关键流程逻辑 (Workflow)

### Step 1: 需求触发

客户通过 [小智](https://sagemro.com/) 发起维修。系统通过 AI 诊断生成初步故障报告，并根据 `service_area` 推送给对应区域的合伙人。

### Step 2: 抢单与交付

合伙人确认接单，系统解锁客户详细地址。 合伙人到达现场，必须按照 **数字化 SOP** 操作：

1. **签到**（地理位置校验）。
2. **初查照片上传**（保留现场证据）。
3. **技术方案调取**（通过小智调取维修参数）。
4. **完工验收单**（客户在合伙人手机端电子签名）。

### Step 3: 结算闭环

1. 客户在线支付或确认账期。
2. 总部后台开具发票发送给客户。
3. 系统自动计算分润：`payout = total * (1 - rate)`。
4. 资金进入合伙人平台钱包，可申请提现至其主体对公账号。

## 5. 核心约束逻辑 (Guardrails)

- **反私单逻辑：** 客户需发票方能入账，合伙人无法提供。维修档案记录在 [我的工单](https://sagemro.com/) 中，脱离平台则失去质保支撑。
- **风控逻辑：** 工单执行中若触发核心部件（如激光源）变动，必须强制请求总部专家视频核验。
- **信用体系：** 引入 `Credit_Score`。发生安全事故或私下切客行为，扣除保证金并取消承包权。

## 7. 业务定义与法律架构

- **模式：** 垂直激光切割维修，总部(SAGEMRO)总包 + 区域合伙人分包。
- **法律关系：** 总部与合伙人为 **B2B 契约关系**，非雇佣劳动关系。
- **资金链：** 客户 $\rightarrow$ 平台 (全额/开票) $\rightarrow$ 合伙人 (结算分润)。

## 8. 合伙人分级与分成模型 (Profit Sharing)

在系统中建立 `PartnerLevel` 模型，动态计算每笔订单的结算：

| **等级** | **标识** | **基础分润比例 (合伙人:平台)** | **技术门槛**                             |
| -------- | -------- | ------------------------------ | ---------------------------------------- |
| **L1**   | Junior   | **65% : 35%**                  | 易损件更换、基础点检                     |
| **L2**   | Senior   | **75% : 25%**                  | 光路调节、系统调试、机械大修             |
| **L3**   | Expert   | **85% : 15%**                  | 激光器源(Laser Source)维修、复杂电路诊断 |

### 核心结算公式：

```
Final_Payout = (Total_Order_Amount - VAT_Tax - Spare_Parts_Cost) * P_Rate + Accessory_Commission - Quality_Deposit
```

- `VAT_Tax`: 根据开票税率预留的税金。
- `Accessory_Commission`: 配件销售额的 10%-15%（如果合伙人带货）。
- `Quality_Deposit`: 每单扣除 5% 存入合伙人保证金池。

## 9. 数字化 SOP 状态机逻辑

合伙人在执行工单时，系统必须强制触发以下状态扭转，否则不予结算：

1. **Matched (已接单):** 合伙人锁定工单，系统推送设备历史维修记录（[我的工单](https://sagemro.com/)数据）。
2. **Arrived (已到达):** GPS 校验，上传机器铭牌照片。
3. **Diagnosed (已诊断):** 调用 [小智助手](https://sagemro.com/) 接口，确认故障点与所需配件。
4. **Executing (维修中):** 按照系统下发的 SOP 步骤，分阶段上传动作凭证（如光路对焦图）。
5. **Accepted (已验收):** 客户在 App 端进行电子签名，系统生成维修报告。
